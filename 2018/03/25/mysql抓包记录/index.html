果断打开浏览器，官方文档搜索，进入了如下页面二进制协议值
找到文档中关于date及datetime类型的协议包

ProtocolBinary::MYSQL_TYPE_DATE, ProtocolBinary::MYSQL_TYPE_DATETIME, ProtocolBinary::MYSQL_TYPE_TIMESTAMP:
如果年月日时分秒和毫秒都为空的话，编码长度为0
如果时、分、秒和毫秒都是0, 则编码的长度是4,为0部分则不发送
如果毫秒为0，长度为7，毫秒部分不发送
其他情况下，长度为11

  • length(1) –后续的字节长度(有效数据为0,4,7,11)
  • year(2) –year
  • month(1) –month
  • day(1) –day
  • hour(1)–hour
  • minite(1) –minutes
  • second(1) -seconds
  • micro_second(4) –micro-seconds

mysql date time 等类型其他相关资料

官方文档关于date和time类型，关于year date time timestamp 和datetime类型的存储需求，如下表所示：

Type Storage before Mysql 5.6.4 Storage mysql5.6.4
year 1 个字节，小端存储 unchanged
date 3 个字节，小端存储 unchanged
time 3 个字节, 小端存储 3 bytes+小数位秒存储，大端存储
timestamp 4 个字节,小端存储 4个字节+小数位秒的存储，大端存储
datetime 8个字节，小端存储 5个字节+小数位秒存储，大端存储

在5.6.4版本之前，数据类型按照以下方式编码：

  • year：一个字节整型
  • date：3个字节整型，按照YYYY1632+MM*32+DD这样的方式计算
  • Time：3个字节的整型，按照DD243600+HH3600+MM60+SS的方式计算
  • Timestamp：4个字节的整型。记录自1970-01-01 00:00:00到当前事件的毫秒数
  • datetime: 8个字节整型。四个字节表示日期 YYYY10000+MM100+DD 四个字节表示时间，HH10000+MM100+SS
    而在5.6.4版本以后，time timestamp和datetime类型包含了分数部分，表示秒的精度。这些类型统一改由大端存储（为了兼容memcmp())。秒的精度紧跟着整数部分。
